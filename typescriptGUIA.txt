Classes
class abstract é um modelo para outra classe
năo é  possivel criar uma instanciia usando  o abstract
Diferença entre PRIVATE e PROTECTED
private restringe o acesso estritamente ŕ própria classe onde foi declarado.
protected permite o acesso pela própria classe e também por suas subclasses (herança), sendo mais flexível para reutilizaçăo de código.

Overload
Overload (sobrecarga) no TypeScript é uma funcionalidade que permite definir múltiplas assinaturas para uma mesma funçăo ou método, mas com apenas uma implementaçăo. Isso possibilita que a funçăo aceite diferentes tipos ou números de argumentos, retornando tipos específicos baseados na entrada, garantindo segurança de tipos (type safety) e clareza no código.

Type guards
Type guards săo técnicas em TypeScript que realizam verificaçőes em tempo de execuçăo
type guards é a verificaçăo dos tipos de um union type
para verificar parametros de funçőes usa-se o 'typeof'
para verificar classes usa-se o instanceof
typeguards: 'in', 'instanceof', 'typeof'

Narrowing
Narrowing (estreitamento) no TypeScript é o processo de refinar um tipo de dado mais amplo (como uma uniăo de tipos string | number) para um tipo mais específico (apenas string ou apenas number) dentro de um bloco de código.
É o processo(uma tecnica) de refinar ou estreitar o tipo de uma variavel

Generics
Os genericos permitem criar funçőes, classes, e interfaces que podem trabalhar com diferentes tipos de maneira flexivel

Record
O record é um tipo utilitariono ts que permiite criar tipagens de objeto chave-valor

As, satisfies
Use satisfies quando:
Criar objetos
Garantir estrutura correta
Evitar bugs silenciosos
Use as quando:
Vocę TEM CERTEZA ABSOLUTA do tipo
Ex: dados vindos de API já validados
Ou algo que o TS năo consegue inferir

Keyof
keyof ? pega as chaves de um tipo ({ string1: "string1", string2: "string2"})
Resultado ? union de strings ( string1 | string2)
Usado pra:
evitar erros
acessar propriedades com segurança
criar funçőes genéricas inteligentes

Never
never é usado para representar valores que nunca podem ocorrer

Readonly
ele serve para definir a tipagem como imutavel (apenas para leitura)

As const
Torna qualquer valor como literal e ele vira imutavel (funciona como um readonly)
Garante que os valores sejam tratados como literais e imutaveis
isso garante a evitar erros

Pick
č um tipo utilitario que permite criar um novo tiipo a partiir de um tipo existente
reduz a repetiçăo dde código
com o pick podemos escolher propriiedades especificas de estruturas (interfaces) 

Omit
é um tipo utilitario no ts que permite a criaçăo de novos tipos a partir de um existente,
porem removendo um ou mais propriedades especificadas
Bem parecido com o Pick

Partial
é um tipo utilitario no ts usado para transformar todas as propriedades de um objeto em propriedade opicionais
muito util para lidar com objetos que NĂO precisam de todas as propriedades de uma interface


Required
é um tipo utilitario no ts usado para transformar todas as propriedades de um objeto em propriedade requiridas (contrario do Partial)
muito util para lidar com objetos que PRECISAM de todas as propriedades de uma interface
propriedade opicionais viram obrigatorias


Exclude
é usado para remover um tipo de uma  uniăo de tipos
é util  quando queremos excluir uma uniăode tipos para criar outra uniăo de tipos






NODE
npm init -y : gera o package.json

npm install -D : para instalar dependencias do node
dependencias usadas: tsx

Montando a estrutura:
1.Criar pasta '.vscode'
2.Criar um aquivo 'settings.json'
No terminal:
3.Verificar a versăo do node 'node -v'
4.Gerar o package.json 'npm init -y'
5.instalar dependencias de dev 'npm install -D typescript tsx @types/node'
6.Criar os script "'build': 'tsc', 'dev': 'tsx src/index.ts', 'start': 'node .'
7.Criar arquivo tsconfig.json
8.Inserir essa configuraçăo(pode mudar) dentro do arquivo:
{
    "compilerOptions": {
        "esModuleInterop": true,
        "skipLibCheck": true,
        "strict": true,
        "lib": ["ESNext"],
        "target": "esnext",
        "module": "nodenext",
        "moduleResolution": "nodenext",

        "rootDir": "src",
        "outDir": "build"
    },
    "include": ["src"],
    "exclude": ["node_modules"]
}
9.Criar pasta e arquivo 'src/index.ts'
10.Colocar um console.log('hello world!) para teste
11.No terminal, usar comando para executar o arquivo 'npm run dev'
12.Para buildar o arquivo 'npm run build'
12.Para executar o projeto buildado 'npm start'
