
OBS: typescript e javascript são a 'mesma' coisa então um código de um, sempre vai dar certo no outro.
A diferena é que o typescript é uma linguagem fortemente tipada, ou seja, é mais robusta e passa mais
segurança para grandes aplicações

any : qualquer tipo de variavel (mas é recomendado nunca ter any no código)

parametros opcionais (que contem interrogação'?') são os ultimos parametros

As interfaces serve para definir a estrutura de um objeto, quais propriedades ele tem,
de que tipo elas são e tambem definir tipos de funções

-A interface é mais usada para definir o tipo de OBJETOS e pode ser usada para classes tambem
-O type é mais usado para COMBINAR interfaces com tipos primitivos do javascript(string,numbers,arrays,classes...etc)

Tuplas são diferentes de arrays
A Tupla é basicamente uma caixa que vai ter determinados itens em posições
'type CalendarDate = [day: number, month: number, year: number]'

Enums (enumrações)
Uma maneira de definir um conjunto de valores nomeados.
Os Enums deixam o código mais legivel

Type intersections &
Combina muitos tipos em um só onde todas as interfaces e metódos juntam

Union type ||
União de types porem só um deles sera valido
Usando esse operador (||) vai ser um ou outro valor, nunca mais de um valor

Classes
class abstract é um modelo para outra classe
não é  possivel criar uma instanciia usando  o abstract
Diferença entre PRIVATE e PROTECTED
private restringe o acesso estritamente à própria classe onde foi declarado.
protected permite o acesso pela própria classe e também por suas subclasses (herança), sendo mais flexível para reutilização de código.

Overload
Overload (sobrecarga) no TypeScript é uma funcionalidade que permite definir múltiplas assinaturas para uma mesma função ou método, mas com apenas uma implementação. Isso possibilita que a função aceite diferentes tipos ou números de argumentos, retornando tipos específicos baseados na entrada, garantindo segurança de tipos (type safety) e clareza no código.

Type guards
Type guards são técnicas em TypeScript que realizam verificações em tempo de execução
type guards é a verificação dos tipos de um union type
para verificar parametros de funções usa-se o 'typeof'
para verificar classes usa-se o instanceof
typeguards: 'in', 'instanceof', 'typeof'

Narrowing
Narrowing (estreitamento) no TypeScript é o processo de refinar um tipo de dado mais amplo (como uma união de tipos string | number) para um tipo mais específico (apenas string ou apenas number) dentro de um bloco de código.
É o processo(uma tecnica) de refinar ou estreitar o tipo de uma variavel

Generics
Os genericos permitem criar funções, classes, e interfaces que podem trabalhar com diferentes tipos de maneira flexivel

Record
O record é um tipo utilitariono ts que permiite criar tipagens de objeto chave-valor

As, satisfies
Use satisfies quando:
Criar objetos
Garantir estrutura correta
Evitar bugs silenciosos
Use as quando:
Você TEM CERTEZA ABSOLUTA do tipo
Ex: dados vindos de API já validados
Ou algo que o TS não consegue inferir

Keyof
keyof ? pega as chaves de um tipo ({ string1: "string1", string2: "string2"})
Resultado ? union de strings ( string1 | string2)
Usado pra:
evitar erros
acessar propriedades com segurança
criar funções genéricas inteligentes

Never
never é usado para representar valores que nunca podem ocorrer

Readonly
ele serve para definir a tipagem como imutavel (apenas para leitura)

As const
Torna qualquer valor como literal e ele vira imutavel (funciona como um readonly)
Garante que os valores sejam tratados como literais e imutaveis
isso garante a evitar erros

Pick
è um tipo utilitario que permite criar um novo tiipo a partiir de um tipo existente
reduz a repetição dde código
com o pick podemos escolher propriiedades especificas de estruturas (interfaces) 

Omit
é um tipo utilitario no ts que permite a criação de novos tipos a partir de um existente,
porem removendo um ou mais propriedades especificadas
Bem parecido com o Pick

Partial
é um tipo utilitario no ts usado para transformar todas as propriedades de um objeto em propriedade opicionais
muito util para lidar com objetos que NÃO precisam de todas as propriedades de uma interface


Required
é um tipo utilitario no ts usado para transformar todas as propriedades de um objeto em propriedade requiridas (contrario do Partial)
muito util para lidar com objetos que PRECISAM de todas as propriedades de uma interface
propriedade opicionais viram obrigatorias


Exclude
é usado para remover um tipo de uma  união de tipos
é util  quando queremos excluir uma união de tipos para criar outra união de tipos


NonNullLable
Muito util para criar novas tipagens a partir de uma determinada tipagem que pode ser nulla ou indefinida
garante que a tipagem não tenhha null ou indefinido


Extract
É muito util quando a gente quer filtrar um tipo complexo pra selecionar apenas os tipos que contem a descrição que passamos como segundo generico
















NODE
npm init -y : gera o package.json

npm install -D : para instalar dependencias do node
dependencias usadas: tsx

Montando a estrutura:
1.Criar pasta '.vscode'
2.Criar um aquivo 'settings.json'
No terminal:
3.Verificar a versão do node 'node -v'
4.Gerar o package.json 'npm init -y'
5.instalar dependencias de dev 'npm install -D typescript tsx @types/node'
6.Criar os script "'build': 'tsc', 'dev': 'tsx src/index.ts', 'start': 'node .'
7.Criar arquivo tsconfig.json
8.Inserir essa configuração(pode mudar) dentro do arquivo:
{
    "compilerOptions": {
        "esModuleInterop": true,
        "skipLibCheck": true,
        "strict": true,
        "lib": ["ESNext"],
        "target": "esnext",
        "module": "nodenext",
        "moduleResolution": "nodenext",

        "rootDir": "src",
        "outDir": "build"
    },
    "include": ["src"],
    "exclude": ["node_modules"]
}
9.Criar pasta e arquivo 'src/index.ts'
10.Colocar um console.log('hello world!) para teste
11.No terminal, usar comando para executar o arquivo 'npm run dev'
12.Para buildar o arquivo 'npm run build'
12.Para executar o projeto buildado 'npm start'
